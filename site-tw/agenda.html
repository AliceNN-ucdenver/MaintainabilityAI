Governance & Golden Rules

  2. Trust but verify ‚Äî never merge code you don't understand
  3. Treat AI like a junior dev ‚Äî guide, review, validate
  4. Isolate AI changes in separate commits/PRs with labels
  5. Document rationale (what/why), not just code
